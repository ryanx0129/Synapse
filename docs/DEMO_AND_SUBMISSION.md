# Demo and Devpost Submission Handoff

## 1. The wow moment

The key moment is not merely seeing a 3D graph.

The wow moment is:

> The learner answers a concept question, the graph updates its estimate of what they know, a downstream gap becomes explainable through weak prerequisites, and Synapse reveals a focused Repair Path.

## 2. Recommended demo dataset

Use the Machine Learning preset.

Suggested flow:

- Start in 3D Galaxy.
- Show Optimization, Neural Networks, Evaluation, and Generalization clusters.
- Switch to 2D Path.
- Select Backpropagation.
- Show the chain-rule formula and page citation.
- Answer correctly.
- Show mastery increase, semantic mastered indicator, and edge pulse.
- Select Vanishing Gradients or Deep Network Training.
- Activate Repair Path.
- Show prerequisite diagnosis.
- Briefly show PDF/Text upload.
- End on session summary.

## 3. Sub-three-minute video structure

### 0:00–0:20 — Problem

“Dense educational material is linear, but understanding is interconnected. Students often reread without knowing which prerequisite is causing the gap.”

### 0:20–0:45 — Product

Show the 3D graph and explain that each node is source-grounded and each color/icon represents estimated learner state.

### 0:45–1:25 — Core learning loop

Switch to 2D, open a concept, show citation and formula, answer the active-recall prompt, and show mastery update.

### 1:25–1:55 — Differentiator

Trigger Repair Path and explain prerequisite-aware risk propagation.

### 1:55–2:20 — Document intelligence

Show upload/paste flow and explain local PDF extraction plus GPT-5.6 structured graph generation.

### 2:20–2:45 — Technology

Mention:

- Codex with GPT-5.6 built and tested the repository.
- GPT-5.6 powers source-grounded structured extraction and rubric verification when configured.
- React/TypeScript, PDF.js, React Flow, Three.js force graph, Zod, Dexie, and learner-model algorithms.

### 2:45–2:58 — Impact

“Synapse does not just generate study materials. It maintains a spatial twin of the learner’s understanding and tells them what to repair next.”

## 4. Required evidence

Preserve:

- Codex session ID or feedback identifier
- Commit history during hackathon period
- Build logs
- Test output
- Screenshots
- Public repository or required private-repo access
- Public YouTube video under three minutes
- License and third-party acknowledgments
